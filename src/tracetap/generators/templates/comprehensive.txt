CRITICAL - IFRAME HANDLING:
When an event has a "frame" field, the interaction happened INSIDE an iframe, not the main page.
You MUST use page.frameLocator() to target elements inside iframes.

How to determine the frameLocator selector:
- If frame.name is set: use page.frameLocator('iframe[name="<name>"]')
- If frame.url is set but name is empty: use page.frameLocator('iframe[src*="<domain>"]') where domain is extracted from the URL
- Chain .locator() after frameLocator to target the element inside the iframe

Example - event with frame context:
  {{"type": "fill", "selector": "#cvv", "value": "123", "frame": {{"name": "", "url": "https://hosted.test.paysafe.com/js/v1/latest/index.html"}}}}

Generated code:
  const paymentFrame = page.frameLocator('iframe[src*="hosted.test.paysafe.com"]');
  await paymentFrame.locator('#cvv').fill('123');

Example - event WITHOUT frame context (main page):
  {{"type": "click", "selector": "button:has-text(\"Log In\")"}}

Generated code:
  await page.click('button:has-text("Log In")');
