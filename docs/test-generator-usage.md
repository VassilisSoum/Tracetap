# Test Generator Usage Guide

This guide explains how to use the TestGenerator to convert correlated recording data into executable Playwright tests.

## Overview

The TestGenerator uses Claude AI to intelligently synthesize Playwright test code from correlated UI events and network traffic. It understands the relationship between user interactions and API calls to generate meaningful, maintainable tests.

## Quick Start

```python
from pathlib import Path
from tracetap.record import RecordingSession
from tracetap.generators import TestGenerator, GenerationOptions

# 1. Load a recording session
session = RecordingSession.load_session("my-session")

# 2. Analyze and correlate events
result = await session.analyze()

# 3. Generate tests
generator = TestGenerator()
test_code = await generator.generate_tests(
    result.correlation_result,
    options=GenerationOptions(
        template="comprehensive",
        output_format="typescript",
        confidence_threshold=0.7
    )
)

# 4. Save to file
Path("generated.spec.ts").write_text(test_code)
```

## Configuration Options

### GenerationOptions

Controls how tests are generated:

```python
@dataclass
class GenerationOptions:
    template: str = "comprehensive"          # Template type
    output_format: str = "typescript"        # Output language
    confidence_threshold: float = 0.5        # Min correlation confidence
    include_comments: bool = True            # Include reasoning comments
    base_url: Optional[str] = None           # Override base URL
    test_name: Optional[str] = None          # Custom test name
```

#### Template Types

- **`basic`**: Simple test structure with minimal assertions
- **`comprehensive`**: Full test with network assertions and error handling
- **`regression`**: Regression-focused tests with variable extraction

#### Output Formats

- **`typescript`**: TypeScript with @playwright/test (recommended)
- **`javascript`**: JavaScript with @playwright/test
- **`python`**: Python with playwright

#### Confidence Threshold

Controls which events are included based on correlation confidence:

- `0.5` (default): Include events with medium+ confidence
- `0.7`: Only high-confidence correlations (stricter, fewer false positives)
- `0.3`: Include low-confidence events (more coverage, potential noise)

## Advanced Usage

### Custom Templates

Create custom templates in `src/tracetap/generators/templates/`:

```
my_template.txt:
---
Generate a Playwright test from the following correlated events.

Output format: {output_format}
Base URL: {base_url}
Confidence threshold: {confidence_threshold}

Events:
{events_json}

Requirements:
- Use descriptive test names
- Add comments explaining correlations
- Include network response assertions
```

Use with:

```python
options = GenerationOptions(template="my_template")
```

### Filtering by Confidence

```python
# Only include high-confidence events
options = GenerationOptions(confidence_threshold=0.8)

# Include all events (no filtering)
options = GenerationOptions(confidence_threshold=0.0)
```

### Multiple Output Formats

```python
# Generate TypeScript version
ts_code = await generator.generate_tests(
    result,
    options=GenerationOptions(output_format="typescript")
)
Path("test.spec.ts").write_text(ts_code)

# Generate Python version
py_code = await generator.generate_tests(
    result,
    options=GenerationOptions(output_format="python")
)
Path("test_spec.py").write_text(py_code)
```

### Custom Base URL

```python
options = GenerationOptions(
    base_url="https://staging.example.com"
)
```

This will replace URL placeholders in the generated code.

## Generated Code Structure

### TypeScript Output

```typescript
// Generated by TraceTap
// Template: comprehensive
// Confidence threshold: 0.7
// DO NOT EDIT - This file is auto-generated

import { test, expect } from '@playwright/test';

test('user login and dashboard navigation', async ({ page }) => {
  // Navigate to login page
  await page.goto('https://example.com/login');

  // Fill login form (confidence: 0.9)
  // Triggers: POST /api/auth/login (200)
  await page.fill('#username', 'testuser');
  await page.fill('#password', 'password123');

  // Click login button
  // Triggers: POST /api/auth/login (200)
  const loginPromise = page.waitForResponse(
    response => response.url().includes('/api/auth/login') && response.status() === 200
  );
  await page.click('#login-button');
  const loginResponse = await loginPromise;

  // Verify login response
  const loginData = await loginResponse.json();
  expect(loginData.token).toBeDefined();

  // Navigate to dashboard (confidence: 0.85)
  // Triggers: GET /api/user/profile (200)
  await page.goto('https://example.com/dashboard');
  await page.waitForSelector('#user-profile');
});
```

### Python Output

```python
"""
Generated by TraceTap
Template: comprehensive
Confidence threshold: 0.7
DO NOT EDIT - This file is auto-generated
"""

from playwright.sync_api import Page, expect

def test_user_login_and_dashboard_navigation(page: Page) -> None:
    # Navigate to login page
    page.goto('https://example.com/login')

    # Fill login form (confidence: 0.9)
    # Triggers: POST /api/auth/login (200)
    page.fill('#username', 'testuser')
    page.fill('#password', 'password123')

    # Click login button
    # Triggers: POST /api/auth/login (200)
    with page.expect_response(
        lambda response: '/api/auth/login' in response.url and response.status == 200
    ) as response_info:
        page.click('#login-button')

    login_response = response_info.value
    login_data = login_response.json()
    assert login_data['token'] is not None

    # Navigate to dashboard (confidence: 0.85)
    # Triggers: GET /api/user/profile (200)
    page.goto('https://example.com/dashboard')
    page.wait_for_selector('#user-profile')
```

## API Reference

### TestGenerator

Main orchestrator for test generation.

```python
class TestGenerator:
    def __init__(self, api_key: Optional[str] = None):
        """Initialize with optional API key (or use ANTHROPIC_API_KEY env var)."""

    async def generate_tests(
        self,
        correlation_result: CorrelationResult,
        options: Optional[GenerationOptions] = None,
    ) -> str:
        """Generate test code from correlation result."""
```

### CodeSynthesizer

Lower-level code synthesis utilities.

```python
class CodeSynthesizer:
    @staticmethod
    def generate_playwright_action(ui_event) -> str:
        """Convert UI event to Playwright action code."""

    @staticmethod
    def generate_network_assertion(network_call, confidence: float) -> str:
        """Convert network call to assertion code."""

    @staticmethod
    def generate_test_name(events: List[CorrelatedEvent]) -> str:
        """Generate descriptive test name from events."""
```

## Error Handling

### Missing API Key

```python
# Raises RuntimeError
generator = TestGenerator()
await generator.generate_tests(result)
# RuntimeError: Claude API client not initialized. Set ANTHROPIC_API_KEY...
```

### Invalid Template

```python
# Raises FileNotFoundError
options = GenerationOptions(template="nonexistent")
await generator.generate_tests(result, options)
# FileNotFoundError: Template not found: nonexistent
```

### Invalid Output Format

```python
# Raises ValueError
options = GenerationOptions(output_format="ruby")
await generator.generate_tests(result, options)
# ValueError: Invalid output format: ruby
```

### Empty Correlation Result

If no events meet the confidence threshold:

```python
options = GenerationOptions(confidence_threshold=0.99)
code = await generator.generate_tests(result, options)
# Generates minimal test with warning comment
```

## Best Practices

### 1. Start with Default Settings

```python
# Simple, works for most cases
test_code = await generator.generate_tests(result)
```

### 2. Tune Confidence Threshold

- High threshold (0.8+): Fewer tests, higher quality
- Medium threshold (0.5-0.7): Balanced approach
- Low threshold (0.3-0.5): Maximum coverage

### 3. Review Generated Tests

Always review and edit generated tests:
- Check selectors are correct
- Verify test data makes sense
- Add custom assertions as needed
- Remove the "DO NOT EDIT" header once reviewed

### 4. Use Comprehensive Template

For production tests, use the comprehensive template:

```python
options = GenerationOptions(template="comprehensive")
```

### 5. Version Control Generated Tests

After reviewing and editing, commit to version control like any other test.

## Troubleshooting

### "Anthropic library not installed"

```bash
pip install anthropic>=0.71.0
```

### "No Anthropic API key found"

```bash
export ANTHROPIC_API_KEY=your_key_here
```

### Generated code has syntax errors

The generator includes basic syntax validation, but edge cases may occur. Common fixes:

1. Check selectors are properly escaped
2. Verify URLs are valid
3. Check for special characters in test data

### Low-quality tests

If generated tests are low quality:

1. Increase confidence threshold: `confidence_threshold=0.8`
2. Try different template: `template="regression"`
3. Review correlation quality in original recording
4. Add more context to templates
